# Operational Patterns & Conventions

Configuration, persistence, output conventions, testing, scan-skip and date discipline.
Siblings: [architecture.md](architecture.md) ¬∑ [integrations.md](integrations.md) ¬∑
[security.md](security.md) ¬∑ [sharp-edges.md](sharp-edges.md).

- **Configuration:** one file, `vault.config.json`, loaded per-invocation by `loadConfig()` with defaulting (`??`) and traversal validation on every dir value. No env vars except `NO_COLOR` (dashboard), `SHELL` (post-scaffold drop-in), and `VAULT_AGENT_CMD` (nightly.sh's agent-runner escape hatch, `eval`'d ‚Äî user-controlled machine, user-set var).
- **Persistence:** markdown files + git. Generated artifacts: `index.md` per folder (header `<!-- generated by scripts/index.ts ... -->`, no frontmatter, validator-skipped by basename), `dashboard/status.txt` (`--write`, ANSI-free), root `log.md` (nightly bullets), `.nightly.log` (cron stdout, gitignored).
- **Logging/observability:** stdout/stderr only. Error convention everywhere: `path:line: msg` (validators, search citations ‚Äî deliberately grep/editor-clickable). Success convention: `‚úì <action> ‚Üí <relpath>`. Exit codes: 0 ok, 1 operation/validation failure, 2 usage error. `--json` flags on validate-vault, search, snapshot for machine consumption.
- **Entry-point wrapper:** every script main goes through `runMain()` (`src/config.ts:60`) ‚Äî clean `error: <msg>` + exit(1), never a raw stack trace.
- **Write safety:** `insertBullet()` (`src/capture.ts:64`) is the canonical write path ‚Äî modify, validate in-process, **restore original bytes and throw** on any validator error. Route new note-mutating features through it (log-turn.ts does; add no parallel insert logic).
- **git discipline:** via `Bun.spawnSync` only; check exitCode, surface stderr in the thrown error. `nightly.ts`/`consolidate.ts` use `git mv` for auditable moves; nothing ever pushes without an explicit `--push`.
- **Testing:** `bun test` only ‚Äî no lint, no typecheck gate. Tests import pure fns directly (`buildIndex`, `bm25`, `buildSnapshot`, `markProcessed`, `validateVaultNote`‚Ä¶) and scaffold temp trees for integration cases. Determinism is a tested property: `buildIndex` output is byte-identical for identical input (no timestamps/randomness/fs-order dependence).
- **Scan-skip discipline:** `NONCONTENT_SUBTREES` (`src/config.ts:36` ‚Äî `.git, node_modules, .obsidian, .claude, .omc, scripts, dashboard`) is shared; search and validate-vault each layer `agents` + `handoffs` on top (search: coordination state isn't recall material; validate: those trees have their own validator). `inSkippedSubtree()` matches ANY path segment, so `projects/x/node_modules/y.md` is skipped like a root-level one. `wiki/raw/` + `wiki/processed/` are schema-exempt in full scans (unedited clippings) but still checked when named explicitly.
- **Date discipline:** `YYYY-MM-DD` strings everywhere; `isValidCalendarDate()` rejects JS-normalized overflow (2026-02-30). In `consolidate.ts`, comparisons are pure calendar-date: `dateOrNull()` parses as UTC midnight and `todayCalendarDate()` recasts local Y-M-D the same way ‚Äî **never compare a parsed date against raw `new Date()`** (off-by-a-day at TZ boundaries).
- **Release/dev flow:** feature branches ‚Üí PR ‚Üí GitHub CI (`bun test`) ‚Üí `chore: release X.Y.Z` commit ‚Üí npm publish. Local dev loop = `task scaffold` + `task dashboard` against `/tmp/vault-init-smoke`.
